# Project Tracking

Эта папка хранит рабочую память проекта: roadmap, задачи, отчеты, решения, чеклисты и шаблоны.

Цель папки - чтобы по репозиторию было понятно не только что написано в коде, но и почему это сделано, что уже проверено, какие вопросы открыты и что делать дальше.

## Публичный режим

Проект публичный. Любые task/report/decision/roadmap файлы должны проходить redaction review.

Не добавлять:

- токены, ключи, пароли, cookies;
- приватные URL, subscription links, VPN/proxy endpoints;
- локальные абсолютные пути пользователя и внутренние hostname/IP;
- приватные v2rayN configs, profile payloads и реальные логи;
- персональные данные.

Использовать плейсхолдеры: `<redacted-url>`, `<local-v2rayn-dir>`, `<subscription-profile>`.

## Структура

```text
project-tracking/
  roadmap/     # этапы и крупные планы
  tasks/       # постановки задач
  reports/     # отчеты по выполнению
  decisions/   # архитектурные, продуктовые и процессные решения
  checklists/  # повторяемые проверки
  templates/   # шаблоны документов
```

## Нумерация

Используем префиксы:

- `0000-...`
- `0001-...`
- `0002-...`

Один номер связывает задачу и отчет:

- `tasks/0004-add-installer-packaging-flow.md`
- `reports/0004-add-installer-packaging-flow-report.md`

## Как работать

1. Перед задачей создать или обновить task-файл.
2. В процессе фиксировать вопросы, решения и изменения scope.
3. После выполнения создать или обновить report-файл.
4. Если появилось устойчивое правило или архитектурное решение, добавить decision-файл.
5. Если меняется процесс проверки, обновить чеклист.
6. Перед коммитом проверить, что публичные документы не содержат приватных данных.

## Минимальная приемка

Работа не считается закрытой, пока не выполнены проверка, отчет и обновление связанных мест: Rust/Tauri backend, React frontend, shared types/API contracts, tests, documentation, build/release scripts, configs, roadmap/tasks/reports.

Beads был исходным трекером для миграции. После переноса актуальный источник истины - Markdown-файлы в этой папке.
