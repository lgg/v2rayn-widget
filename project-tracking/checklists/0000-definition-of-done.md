# 0000 - Definition of Done Checklist

Использовать перед закрытием любой задачи.

## Task Understanding

- [ ] Понятна цель задачи.
- [ ] Понятны критерии приемки.
- [ ] Понятен scope и out of scope.
- [ ] Открытые вопросы зафиксированы.
- [ ] Блокирующие вопросы уточнены до реализации.

## Implementation

- [ ] Реализация соответствует задаче.
- [ ] Изменения минимально достаточные.
- [ ] Не сделаны лишние побочные рефакторы.
- [ ] Не сломана существующая архитектура.

## Consistency

- [ ] Rust/Tauri backend обновлен, если затронута backend-логика.
- [ ] React/TypeScript frontend обновлен, если затронут пользовательский сценарий.
- [ ] Shared types / API contracts обновлены, если менялся контракт.
- [ ] Tests обновлены или причина отсутствия тестов зафиксирована.
- [ ] Документация обновлена.
- [ ] Roadmap/task/report обновлены.
- [ ] Examples/env/config обновлены, если менялись настройки.
- [ ] Build/release scripts обновлены, если менялся запуск, сборка или релиз.

## Desktop Release

- [ ] Portable build flow не сломан.
- [ ] Installer flow проверен или явно отмечен как не проверенный.
- [ ] Изменения Windows privileges/user context описаны, если затронуты.
- [ ] Реальные v2rayN configs/logs/subscription data не попали в репозиторий.

## Tests and Checks

- [ ] Lint/static checks выполнены, если есть.
- [ ] Tests выполнены, если есть.
- [ ] Build выполнен, если есть.
- [ ] Ключевой пользовательский сценарий проверен.
- [ ] Ошибки/edge cases проверены там, где это важно.
- [ ] Документация и config review выполнены.

## Public Data Review

- [ ] Нет токенов, API keys, паролей, cookies, приватных ключей.
- [ ] Нет приватных URL, subscription links, VPN/proxy endpoints, webhook URLs.
- [ ] Нет локальных абсолютных путей пользователя, внутренних hostname/IP или сетевых шар.
- [ ] Нет приватных v2rayN configs, profile payloads и реальных логов.
- [ ] Нет персональных данных.

## Report

- [ ] Отчет создан или обновлен.
- [ ] В отчете описано, что сделано.
- [ ] В отчете перечислены измененные файлы/области.
- [ ] В отчете описаны проверки и результаты.
- [ ] В отчете указано, что не удалось проверить.
- [ ] Остаточные риски и следующие шаги зафиксированы.
